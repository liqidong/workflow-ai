# Contributing

这个仓库记录 AI 编程薄工作流。贡献时请保持它薄。

## 贡献边界

- 示例保持通用。
- 不加入个人路径、私有项目名、客户数据、凭据或机器专属说明。
- 优先写短说明，不写框架代码。
- 不引入 scheduler、ledger、lock service 或后台自动化，除非已有强证据证明必要。
- 如果新增规则，也要说明什么时候不该用。

## Review Checklist

- 是否保留一个 source of truth。
- 是否避免复制外部工具内部流程。
- 是否帮助用户用更好证据收尾。
- 用户是否能 clone 后直接阅读使用，而不需要知道作者机器或私有项目。
